Agenda

  • Sync / Async of JS
  • Debounce : Debouncing accepts a function and transforms it in to an updated (debounced) function so that the code inside the original function is executed after a certain period of time. If the debounced function is called again within that period, the previous timer is reset and a new timer is started for this function call. The process repeats for each function call. Eg : Searchbox, on scroll user keeps scroll and won't stop so you wont be able to trigger activity until user stop scroll. - Can't use Debounce.
  • Throttle : If events exists then it will execute after dalay/timebox. Eg. timebox 4 seconds : Event1 _ _ _ _ Event2 _ _ _ _ Event3 etc... given. Eg : Scroll
  • Debounce vs Throttle
