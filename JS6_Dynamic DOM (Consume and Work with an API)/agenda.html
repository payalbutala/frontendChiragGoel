Agenda

  • JSON : JavaScript Object Notation
    1. Looks like object with key pair value. Key in double quotes.
    2. Value : String, number, object, array, boolean, null. [Invalid Values for json but valid for object: undefined, function, symbols, date ]
    3. To communicate between 1 machine & another JSON should be converted to string by method JSON.stringify(obj) [Serialization] and convery back to JSON using JSON.parse(str)
    4. API : get & send data form server to client, helps to communicate between 2 machines (client and server)
    5. endpoint = URL
    6. API -> call api method provided by browser - fetch which will request & you will get response. Three things happens in response :
      1. wait
      2. success
      3. failure
    7. Promise helps to do chining
      1. then method after request if its request fulfilled/success - execute
      2. catch method after request if its request error/falied - caught
